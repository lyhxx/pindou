# 拼豆工坊

拼豆工坊是一个本地运行的拼豆图纸生成与编辑工具，用来把图片转换成可直接开工的拼豆图纸，并支持在浏览器里继续修图、整理画布、导出图纸与工程文件。

[在线体验](https://pindou.javai.cn) · [更新日志](./CHANGELOG.md) · [开发 Skill](./skills/pindou-dev/SKILL.md)

## 项目摘要

- 产品定位：本地运行的拼豆图纸工作台，不依赖后端服务
- 核心能力：图片转拼豆图、手工修图、颜色替换、图纸导出
- 适用场景：拼豆图纸制作、像素图转拼豆、Perler beads pattern 编辑

## 项目定位

- 本地处理：不依赖后端，图片、画布、图纸转换都在浏览器内完成
- 拼豆图纸编辑：支持画布创建、图片生成图纸、手工修图、颜色替换、导出
- 工作台式交互：三栏编辑器布局，画布在中间独立舞台内操作

## 为什么是这个项目

- 不走云端上传，图片定位、图纸生成、修图和导出都在本地浏览器完成
- 不是一次性转换器，生成图纸后还能继续手工修正颜色、网格和留白
- 首页和编辑器都收口成工具站工作流，打开后可以直接进入制作

## 架构说明

- 状态管理集中在 `src/features/editor/editorStore.ts`
- 图纸生成逻辑集中在 `src/features/editor/quantizeImage.ts`
- 中间画布舞台由 `CanvasStage` 负责渲染与交互
- 图片定位预览与正式图纸生成分离，避免调整图片参数时直接破坏当前图纸
- 当前版本只保留一份本地缓存工程，不再维护多项目列表
- 编辑器帮助说明统一收口在 `src/features/editor/helpContent.ts`，不要把帮助文案重新散写回各个组件

## 后续开发计划

- 撤销 / 重做稳定性验证：补齐连续绘制、批量替换、裁切空白、留白 4 格、重新生成图纸等操作的回退测试，确保历史栈行为稳定。
- 图纸打印版优化：继续优化导出图纸的页眉、坐标区、颜色清单布局和打印可读性，减少大图导出后的信息拥挤问题。
- 颜色替换工作流优化：补强从画布吸色、从颜色统计快速定位源色、再批量替换目标色的连续操作路径，减少手动查找成本。
- 图纸质量提示：在生成结果旁补充颜色数量、孤立色块、过碎区域等基础指标，帮助用户判断当前图纸是否还需要继续修图。
- 品牌 / 库存约束色库：在现有调色板之上扩展品牌、系列、库存和停产状态，为后续“按真实可买颜色生成图纸”做准备。
