# 拼豆网站页面线框方案

## 1. 文档目标

本线框方案直接为真实编辑器实现服务，重点定义：

- 页面结构
- 模块优先级
- 交互位置
- 信息密度

不再讨论抽象概念页。

---

## 2. 页面范围

首版只围绕两页展开：

1. 首页
2. 编辑器页

其中编辑器页优先级最高。

---

## 3. 首页线框

## 3.1 首页目标

首页只承担三个任务：

1. 进入编辑器
2. 新建项目
3. 打开已有项目

## 3.2 首页结构

### 顶部

- Logo / 产品名
- 最近项目入口
- 进入编辑器按钮
- 新建项目按钮

### 首屏主体

左侧：

- 一句话定位
- 两个主按钮
- 简短能力说明

右侧：

- 参考成品或图纸视觉样例

### 下方

- 快速入口卡片
- 项目列表

首页无需复杂营销内容。

---

## 4. 编辑器总线框

## 4.1 总结构

```text
+--------------------------------------------------------------+
| 顶部状态栏                                                   |
+---------+----------------------------------------+-----------+
| 左栏    | 中间图纸舞台                           | 右栏      |
|         |                                        |           |
|         | 工具栏                                 |           |
|         | ------------------------------------   |           |
|         | 坐标尺 + 图纸画布                      |           |
|         |                                        |           |
|         |                                        |           |
+---------+----------------------------------------+-----------+
| 底部状态栏                                                   |
+--------------------------------------------------------------+
```

## 4.2 工作台约束

- 整个编辑器锁定在 viewport 内
- 顶栏固定高度
- 底栏固定高度
- 左右栏内部滚动
- 中间舞台独立裁切
- 页面本身不滚动

---

## 5. 顶部状态栏线框

左侧：

- 返回首页
- 项目名

中间：

- 自动保存状态
- 画布尺寸

右侧：

- 新建项目
- 切换项目

要求：

- 单行
- 扁平
- 高度紧凑

---

## 6. 左栏线框

左栏按照流程顺序排列。

## 6.1 项目管理

- 当前项目
- 重命名
- 新建
- 删除
- 项目列表

## 6.2 画布

- 宽度
- 高度
- 扩边
- 裁切
- 居中重排

## 6.3 图片

- 上传图片
- 替换图片
- 图片信息

## 6.4 定位

- 缩放滑杆
- X 偏移
- Y 偏移
- 重置

## 6.5 转换

- 抖动模式
- 去背景
- 容差
- 重新生成

布局要求：

- 每区块标题小而清晰
- 控件紧凑堆叠
- 同屏尽量看到更多区块

---

## 7. 中间图纸舞台线框

## 7.1 工具栏

左侧工具组：

- 画笔
- 橡皮
- 吸色

中间历史组：

- 撤销
- 重做

右侧视图组：

- 显示网格
- 缩放百分比
- 重置视图

## 7.2 主舞台

结构：

```text
+--------------------------------------------------+
| 顶部列坐标尺                                     |
|   1 2 3 4 5 6 7 ...                              |
|                                                  |
| 左侧行坐标尺 | 图纸网格区                        |
| 1            | [F5][F5][F5][  ][H2]...          |
| 2            | [F5][F5][F5][A2][H2]...          |
| 3            | ...                               |
|                                                  |
+--------------------------------------------------+
```

## 7.3 图纸区细节

必须包含：

- 白底
- 正方格
- 每格色号
- 每 10 格粗线
- hover 高亮格
- 当前绘制格反馈

## 7.4 图层逻辑

用户能感知到的只有一个图纸区，但内部要明确分层：

- 底图
- 格子颜色
- 色号文字
- 坐标尺
- 光标层

---

## 8. 右栏线框

## 8.1 当前颜色

- 大色块
- 色号
- 名称

## 8.2 紧凑色板

建议用矩阵或紧凑清单：

- 色块
- 色号
- 启用状态

不是一行一个大卡。

## 8.3 颜色统计

- 色块
- 色号
- 数量
- 占比

## 8.4 项目信息

- 画布尺寸
- 成品尺寸
- 总格数
- 使用颜色数

## 8.5 导出

- 导出图纸 PNG
- 导出成品 PNG
- 导出工程 JSON
- 导出颜色清单

---

## 9. 底部状态栏线框

左侧：

- 当前工具

中间：

- 当前坐标
- 当前格色号

右侧：

- 当前缩放
- 当前视图偏移

底栏必须是低干扰但实用的信息区。

---

## 10. 图纸模式交互线框

## 10.1 默认状态

- 鼠标移动到格子上，显示 hover 框
- 底栏显示当前格坐标和色号

## 10.2 绘制状态

- 左键点按改单格
- 左键拖动连续绘制

## 10.3 吸色状态

- 右键点击直接吸色
- 或 Alt + 左键吸色

## 10.4 平移状态

- 空格按住后拖动
- 中键拖动

## 10.5 缩放状态

- 滚轮围绕指针缩放
- `0` 回到默认视图

---

## 11. 信息密度要求

这一版线框的判断标准不是“看起来舒服”，而是：

- 常用操作是否在首屏
- 图纸是否足够大
- 是否减少无效空白
- 是否降低切换成本

如果一个线框方案导致：

- 图纸区太小
- 色板区太高
- 左右栏太松

那就不是合格方案。

---

## 12. 组件拆分指向

首页建议拆分：

- `HomeHeader`
- `HomeHero`
- `ProjectList`
- `QuickActions`

编辑器建议拆分：

- `EditorTopbar`
- `EditorSidebarLeft`
- `EditorToolbar`
- `EditorStage`
- `EditorRulerLayer`
- `EditorGridCanvas`
- `EditorStatusbar`
- `EditorSidebarRight`

图纸相关必须继续细拆：

- `GridBackgroundLayer`
- `GridColorLayer`
- `GridTextLayer`
- `GridOverlayLayer`

这不是过度设计，而是为了后续性能优化。
