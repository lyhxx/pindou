# 拼豆网站方案设计

## 1. 目标重定义

本项目不再定义为“图片上传后生成一个好看的拼豆预览页”，而是定义为：

`一个可实际生产使用的拼豆图纸编辑器`

首版必须满足以下真实编辑场景：

- 用户创建任意尺寸画布
- 上传图片后自动转换成拼豆图纸
- 图纸是方格化、带坐标、可编辑、可导出的
- 用户能像使用像素编辑器一样在图纸上继续修稿
- 缩放、平移、绘制、吸色都围绕图纸编辑效率设计
- 页面本身不滚动，编辑器始终是一个完整工作台

这意味着后续方案必须优先服务以下 4 件事：

1. 图纸可读
2. 编辑高效
3. 转换结果可控
4. 导出结果可直接用于照图制作

---

## 2. 当前方向修正

上一轮实现虽然打通了上传、量化、导出和多项目存储，但编辑器形态偏向“演示型工作台”，与真实拼豆图纸编辑器不一致，主要问题如下：

- 中间舞台偏成品预览，不是图纸模式
- 画布不是纯白图纸背景
- 单元格未显示编号与色号
- 没有顶部和左侧坐标尺
- 交互仍是工具切换式，不够像专业绘图工具
- 右栏颜色区信息密度太低
- 左右栏控件尺寸太大，浪费屏幕

本方案从这一版开始明确切换为：

`图纸优先，而不是珠子预览优先`

---

## 3. 首版真实编辑器范围

## 3.1 必做功能

### 画布

- 新建画布
- 自定义宽高，单位为格
- 支持后续扩边、裁切、居中重排
- 支持大画布分区辅助线

### 图片导入与自动转换

- 本地上传图片
- 上传后显示原图叠加预览
- 调整原图缩放与位置
- 一键自动转换为拼豆图纸
- 支持重新生成

### 图纸模式

- 纯白图纸背景
- 正方形网格
- 顶部和左侧编号尺
- 支持每格显示色号
- 每 N 格显示强调分割线
- 支持空白格

### 编辑能力

- 左键绘制
- 左键拖动连续绘制
- 右键或 Alt 吸色
- 擦除
- 滚轮缩放
- 空格拖拽或中键拖拽平移
- 撤销 / 重做

### 调色与转换控制

- 固定内置色板
- 可启用 / 停用指定颜色
- 量化只使用启用颜色
- 抖动开关
- 纯色背景去除

### 导出

- 图纸 PNG
- 成品预览 PNG
- 工程 JSON
- 颜色统计清单

### 存储

- 多项目本地保存
- 项目切换
- 项目重命名
- 项目删除

## 3.2 首版不做

- AI 抠图
- 多图层
- 复杂区域选区工具
- 在线协作
- 后台服务
- 手机端重度编辑

---

## 4. 核心产品原则

## 4.1 图纸第一

用户最终需要的是“照着做”的图纸，不是一个圆珠效果展示图。因此默认主视图必须是图纸视图。

## 4.2 编辑优先于装饰

所有视觉决定都要让位于：

- 格子更清楚
- 色号更清楚
- 工具更高效
- 面板更紧凑

## 4.3 画布是主角

中间区域必须是编辑核心，左右栏只是辅助，不得抢占主要空间。

## 4.4 交互尽量接近专业工具

参考 draw.io、Photoshop、像素编辑器的习惯：

- 左键编辑
- 滚轮缩放
- 空格平移
- 快捷键明确

而不是频繁切换平移工具。

---

## 5. 真实编辑器信息架构

## 5.1 页面结构

项目首版只保留两类主页面：

1. 首页 / 项目入口
2. 编辑器页

其中编辑器页是绝对核心。

## 5.2 编辑器结构

编辑器采用：

`顶部状态栏 + 左侧参数栏 + 中间图纸舞台 + 右侧色板与导出栏 + 底部状态栏`

### 顶部状态栏

- 当前项目名
- 自动保存状态
- 画布尺寸
- 新建 / 切换 / 返回

### 左侧参数栏

- 项目管理
- 画布设置
- 图片上传
- 图片定位
- 转换参数
- 生成操作

### 中间图纸舞台

- 工具栏
- 坐标图纸画布
- 缩放平移视图
- 图纸编辑主区域

### 右侧信息栏

- 当前颜色
- 紧凑色板
- 颜色统计
- 项目信息
- 导入导出

### 底部状态栏

- 当前工具
- 当前坐标
- 当前缩放
- 当前格子色号

## 6. 图纸模式定义

## 6.1 为什么必须切到图纸模式

拼豆编辑器不是普通图片编辑器。它最终交付的是“离散格子说明图”，因此核心视图必须显式展示：

- 行列关系
- 每格颜色归属
- 分区阅读结构

如果只显示圆珠预览，用户难以逐格校对。

## 6.2 图纸视图构成

图纸视图由以下元素组成：

1. 纯白主底
2. 浅灰细网格
3. 橙色粗分区线
4. 顶部列编号
5. 左侧行编号
6. 单元格内部色号文字
7. 当前 hover 高亮
8. 当前编辑光标框

## 6.3 单元格显示规则

每个格子是一个正方形单元。

显示优先级：

1. 颜色填充
2. 色号文字
3. 网格线
4. hover / 选中框

默认文字内容为调色板编号，如：

- `F5`
- `H2`
- `A26`

## 6.4 分区规则

为提升大图纸可读性，默认每 10 格使用一条强调分割线。

例如：

- 细网格：每格
- 粗分割线：每 10 格

这和参考图一致，也方便打印后人工制作。

---

## 7. 图片上传的真实作用

图片上传不是展示层，不是单纯给用户“看一眼”。

它在系统里的真实作用是：

1. 作为量化输入源
2. 作为原图定位源
3. 作为重新生成时的基础数据

首版正确流程应是：

1. 用户上传图片
2. 用户调整图片在画布中的缩放和偏移
3. 用户点击生成
4. 系统将图片采样到网格尺寸
5. 系统将每格颜色映射到拼豆色板
6. 输出为带颜色索引的 `BeadGrid`

不是“上传图只做预览”。

---

## 8. 编辑交互重构原则

## 8.1 目标交互

默认交互切换为：

- 左键：绘制当前颜色
- 左键拖动：连续绘制
- 右键：吸色
- `Alt + 左键`：吸色
- `E`：擦除模式
- 滚轮：以鼠标点为中心缩放
- 空格按住拖动：平移
- 中键拖动：平移

## 8.2 工具栏角色变化

工具栏仍保留，但角色从“必须靠切换才能操作”改为“显示当前模式并提供显式切换入口”。

也就是说：

- 平移不再必须切换工具
- 画笔是默认主模式
- 工具栏更多是状态控制，不是操作门槛

## 8.3 快捷键

首版应明确支持：

- `B`：画笔
- `E`：橡皮
- `I`：吸色
- `Space`：临时平移
- `Ctrl/Cmd + Z`：撤销
- `Ctrl/Cmd + Shift + Z`：重做
- `0`：重置视图
- `G`：显示/隐藏网格

## 9. 数据结构要求

## 9.1 画布模型

```ts
type CanvasSize = {
  width: number;
  height: number;
};
```

## 9.2 网格模型

```ts
type BeadGrid = {
  width: number;
  height: number;
  cells: Uint16Array;
};
```

约定：

- `cells.length = width * height`
- `65535` 表示空白格
- 单元格值存调色板索引，不直接存颜色字符串

## 9.3 图纸视图状态

```ts
type StageViewport = {
  scale: number;
  offsetX: number;
  offsetY: number;
};
```

## 9.4 原图定位状态

```ts
type ImageTransform = {
  scale: number;
  offsetX: number;
  offsetY: number;
};
```

必须保持两套状态分离：

- `ImageTransform` 只影响重新生成
- `StageViewport` 只影响查看和编辑

## 9.5 项目状态

项目必须持久化以下信息：

- 项目名
- 画布尺寸
- 原图数据
- 原图定位
- 图纸网格
- 转换参数
- 启用色板集合
- 当前视图
- 当前活动颜色

---

## 10. 渲染层要求

## 10.1 主渲染方式

必须使用 `Canvas 2D` 渲染图纸，不使用 DOM 格子。

原因：

- 大画布 DOM 节点数过高
- 拖拽缩放性能差
- 绘制色号和分区线会导致过多重排

## 10.2 建议渲染分层

图纸渲染按逻辑分层：

1. 背景层
2. 单元格填色层
3. 网格线层
4. 色号文字层
5. hover / 当前光标层
6. 坐标尺层

## 10.3 重绘策略

不同操作重绘范围不同：

- 缩放 / 平移：整层重绘
- 单点绘制：局部重绘优先
- 批量拖绘：脏矩形合并重绘
- 显示隐藏色号：文字层重绘

---

## 11. 调色板与量化策略

## 11.1 首版色板策略

首版使用固定内置色板，不开放自定义导入。

## 11.2 启用 / 停用颜色

右侧色板允许用户停用某些颜色。

规则：

- 停用色不会参与自动量化
- 手动绘制默认只允许使用启用色
- 若当前颜色被停用，自动切到第一个启用色
- 不允许出现 0 个启用色

## 11.3 量化流程

```text
原图 -> 缩放定位 -> 采样到目标网格 -> 去背景 -> 颜色映射 -> 可选抖动 -> 输出网格
```

首版维持：

- 最近邻颜色匹配
- Floyd-Steinberg 抖动
- 纯色背景去除

---

## 12. 导出定义

## 12.1 图纸 PNG

图纸导出必须接近工作区图纸视图，而不是仅导出彩色块。

建议包含：

- 白底
- 网格
- 色号
- 坐标尺
- 粗分区线

## 12.2 成品 PNG

成品 PNG 可作为预览图，保留珠子效果。

## 12.3 工程 JSON

工程文件必须可完整恢复编辑器状态，包括：

- 网格
- 图片源
- 图片定位
- 色板启用状态
- 当前画布尺寸
- 转换参数

## 12.4 颜色清单

颜色清单导出内容：

- 色号
- 名称
- 十六进制颜色
- 数量
- 占比

---

## 13. 布局与滚动硬约束

编辑器必须满足：

- 整页高度等于 viewport
- `body` 不允许纵向滚动
- 左右栏各自内部滚动
- 中间舞台独立裁切
- 画布放大不能撑高页面

这是实现约束，不是建议。

---

## 14. 开发阶段改造顺序

接下来不建议再做零散修补，必须按真实编辑器主链路改造。

### 阶段 1：图纸舞台替换

- 中间舞台从珠子视图切到图纸视图
- 纯白背景
- 方格
- 编号尺
- 色号文字
- 粗分区线

### 阶段 2：交互重构

- 左键画
- 滚轮缩放
- 空格平移
- Alt / 右键吸色
- 减少工具切换依赖

### 阶段 3：右栏压缩与色板重排

- 调色板改为紧凑矩阵或紧凑列表
- 当前颜色与颜色统计信息密度提升
- 按钮尺寸缩小

### 阶段 4：导出图纸升级

- 图纸 PNG 输出编号、网格、分区线
- 与工作区图纸模式一致

### 阶段 5：画布操作增强

- 扩边
- 裁切
- 居中重排
- 大图纸阅读优化

---

## 15. 结论

这个项目接下来不应该再以“网页工具页”的思路推进，而要明确按“拼豆图纸编辑器”推进。

后续所有实现判断标准只有一个：

`用户能不能真的拿这张图纸去做拼豆，并且在编辑器里高效修稿。`

只要不服务这个目标的设计，都应该让位。
