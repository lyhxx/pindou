# 拼豆网站开发交接计划

更新时间：2026-05-25

## 1. 当前项目状态

当前项目是一个纯前端拼豆编辑器原型，技术栈为：

- `Vite`
- `React 19`
- `TypeScript`
- `Zustand`

目前已经从“概念型三栏工作台”开始切向“真实拼豆图纸编辑器”，但还没有完成整轮重构，当前属于：

`主舞台方向已切对，编辑器体验仍在过渡中`

## 3. 今天新增并已落地的关键改动

这轮不是只改文档，已经开始直接改编辑器本体。

## 3.1 图纸舞台已切换方向

`src/features/editor/components/CanvasStage.tsx` 已从原来的圆珠预览思路，切成图纸模式主舞台，已具备：

- 白底图纸区
- 顶部列号
- 左侧行号
- 正方格
- 每 10 格粗分区线
- 格内色号文字
- hover 高亮框

## 3.2 交互已部分重构

当前已接入：

- 滚轮缩放
- 空格按住平移
- 中键平移
- 右键吸色
- `Alt` 吸色
- 左键绘制
- 左键拖动连续绘制

说明：

- 工具栏仍保留
- 但平移已不再必须依赖单独切工具

## 3.3 画笔可用性已修正

`paintCell` 已调整为：

- 即使当前没有 `beadGrid`
- 也可以先创建空网格再开始绘制

这解决了“画笔点了没反应”的核心问题之一。

## 3.4 UI 密度已压缩一轮

已调整：

- 顶栏高度
- 按钮尺寸
- 面板 padding
- 左右栏宽度
- 紧凑色板布局

目标是从“展示型后台页面”向“工作型编辑器”靠拢。

## 4. 当前明确存在的问题

下面这些问题是现在还没完成或完成度不足的，不应误判为已收尾。

## 4.1 图纸导出还不是真正的打印图纸

虽然有图纸 PNG 导出，但目前更接近舞台位图导出，仍不足以完全等同于参考图里的正式图纸输出。

缺少强化项：

- 更稳定的编号尺导出
- 更明确的分页 / 打印导出策略
- 图纸导出规范与编辑区完全一致的校对

## 4.2 底部状态栏信息还不够实用

目前底栏有基本提示，但还没有做到真正生产可用的状态反馈，例如：

- 当前 hover 坐标
- 当前格色号
- 当前格名称
- 当前编辑状态更精确反馈

## 4.3 图纸舞台仍有继续精修空间

当前舞台方向已经对了，但还需要继续打磨：

- 坐标尺样式还可以更接近参考图
- 网格细线与粗线比例还可以更精确
- 格内文字显示阈值可以更合理
- 大画布场景要继续观察清晰度

## 4.4 左右栏虽然缩紧了，但还不够“专业工具级”

当前比上一版已经好很多，但仍可继续优化：

- 标题层级还可以更扁平
- 区块信息密度还能进一步提高
- 色板区还可继续更像 PS / 像素编辑器

## 4.5 扩边 / 裁切 / 居中重排还没做

这是后续真实使用中非常关键的一块，目前尚未开始。

## 4.6 背景去除能力仍然偏 MVP

目前只是：

- 四角采样
- 纯色背景容差去除

对复杂背景不够。

## 4.7 代码里仍需要继续做一次系统清扫

虽然大部分乱码已经处理，但项目是多轮演进下来的，明天继续开发前建议再做一次快速扫查，重点看：

- 编辑器文案
- 导出逻辑
- 右栏统计与提示文案

## 5. 当前 UI 明确存在的问题

这部分是今天新补充的，明天不要再重新判断，直接按问题清单处理。

## 5.1 UI 混了两套语言

当前页面属于：

- 中间舞台已切成图纸编辑器
- 外层壳子仍保留旧的卡片展示页风格

结果是：

- 画布像图纸
- 顶栏和面板像展示型后台
- 整页视觉语言不统一

## 5.2 基础组件仍是“通用卡片 + 通用按钮”

当前 `Button / PanelCard / Topbar` 还没有彻底重构成编辑器专用组件，只是做了局部压缩。

问题表现：

- 工具栏按钮仍带明显 CTA 感
- 面板仍是大卡片堆叠逻辑
- 编辑器没有形成真正的高密度参数面板体系

## 5.3 左右栏结构还偏展示型

虽然内容已经换成编辑器内容，但结构仍然是：

- 一个个独立卡片纵向堆叠
- 区块之间视觉太像内容卡，而不像工具分组

这会导致：

- 页面显得碎
- 视觉层级多余
- 编辑器壳层不够专业

## 5.4 色板区仍不够工具化

当前色板虽然已经压成双列，但还不是成熟的编辑器色板样式。

问题：

- 还偏卡片项
- 状态切换仍显碎
- 不像 PS / 像素编辑器的高频色板区

## 5.5 样式变量更新了，但整体系统没有完全落地

`tokens.css` 已经开始向高密度工具风调整。

但 `globals.css` 仍保留较多旧工作台样式：

- 圆角逻辑
- 卡片背景逻辑
- 面板阴影逻辑
- 外层工坊感过重

结论：

`当前 UI 不是单点乱，而是处在“新图纸舞台 + 旧外壳体系”的混搭状态。`

## 6. 当前完成度判断

如果按目标“真实拼豆图纸编辑器”来衡量，当前大致可以这样看：

## 6.1 已经可用的部分

结论：

`现在是一个可运行、可继续验证方向的编辑器原型，不是最终可交付版本。`

## 7. 明天建议开发顺序

不要散着修，建议严格按下面顺序继续。

## P1：图纸导出升级

优先级最高。

目标：

- 让导出的图纸 PNG 尽量接近参考图逻辑
- 输出坐标尺
- 输出格内色号
- 输出粗分区线

原因：

- 当前主舞台已切到图纸模式
- 导出能力必须跟上，否则“可编辑”但“不可生产”

## P2：底部状态栏补全

目标：

- 显示当前 hover 坐标
- 显示当前格色号
- 显示当前格名称
- 显示视图缩放值

原因：

- 这会显著提升修稿效率

## P3：UI 基础组件收口

目标：

- 重做 `Button`
- 重做 `PanelCard`
- 重做 `Topbar`

要求：

- 分离 CTA 按钮、工具按钮、面板按钮
- 分离展示卡片与编辑器面板
- 统一编辑器外层视觉语言

## P4：左右栏 UI 重构

目标：

- 左栏从“大卡片堆叠”改成“紧凑参数面板”
- 右栏从“卡片展示”改成“高密度信息区”
- 降低外壳装饰性

## P5：色板区重构

目标：

- 改成真正编辑器风格的紧凑色板区
- 当前色独立展示
- 色板矩阵或更紧凑的工具列表

## P6：舞台细节打磨

目标：

- 微调网格视觉
- 微调坐标尺视觉
- 微调文字显示阈值
- 确认不同画布比例下显示效果

## P7：画布高级操作

目标：

- 扩边
- 裁切
- 居中重排

这是后续必须补上的真实编辑能力。
